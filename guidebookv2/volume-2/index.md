# 卷二｜runtime core 主线

卷二回答：一条请求怎么真正进入 Codex 的 runtime core，并最终闭合成一轮可以持续推进的工作回合。

## 本卷读完后你应该得到什么

1. 知道请求怎样从入口真正挂进 runtime core 主线
2. 知道 `ThreadManager`、`CodexThread`、`Codex` 怎样接成主工作链
3. 知道动作结果为什么会重新回到当前 turn，而不是另起一条线
4. 知道一轮工作回合怎样继续，以及怎样收口

## 卷二八步地图

- **01** 请求怎么正式进入 runtime 主线
- **02** `ThreadManager`、`CodexThread`、`Codex` 怎么接成主工作链
- **03** 当前工作面怎么被组织出来
- **04** thread / turn 为什么是同一条工作线上的不同层次
- **05** 系统为什么会在这一轮进入能力路径
- **06** 动作结果怎么重新回流当前工作回合
- **07** 一轮工作回合什么时候继续、什么时候收口
- **08** 怎样把整条 runtime core 主线重新压成稳定运行图

## 本卷最后要留下的一句话

> **Codex runtime core 不是一张对象表，而是一条会不断判断、必要时进入动作、再把结果回流当前 turn 的工作回合主线。**
