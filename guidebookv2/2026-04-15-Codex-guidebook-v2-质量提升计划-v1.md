## 二、下一阶段总策略

下一阶段按下面这个原则推进：

> **认知递进 > 卷级收口 > 图层统一 > 入口与发布质量。**

## 三、三阶段执行方案

# 阶段 1：图层统一收口

**目标：** 把当前已落下的图，从“局部解释图”升级成“全书统一视觉解释层”。

## 1.1 任务

### A. 统一图的标题语法
统一为更短、更稳、更适合手机端的标题写法。

建议统一原则：
- 少用过长并列句
- 一张图只保留一个主判断
- 标题尽量做到“图名即结论”

例如：
- 不是：`approval、policy、sandbox、route：都在执行前主链里合流`
- 可以考虑压成：`执行前主链：approval、policy、sandbox、route`

### B. 统一图的版式
统一这些元素：
- 标题位置
- 副标题长度
- 容器边距
- 颜色语义
- 箭头风格
- 底部收口句样式

### C. 统一图的阅读顺序
每张图尽量都能对应下面四类之一：
- 全景图
- 主链图
- 边界图
- 对照图

不要让同类图在结构上差异太大。

### D. 专门做移动端适配
重点处理：
- 右侧过挤
- 标题过长
- 一行字过宽
- 底部收口句过密

### E. 建立图资产清单
建议补一份图索引，列出：
- 文件名
- 对应章节
- 图类型
- 图要回答的主问题
- 是否已移动端优化

## 1.2 产出物

- 一批统一风格后的 SVG
- 一份图资产索引文档
- 一轮正文中图前导语的同步微调

## 1.3 完成标准

- 现有 14 张图标题风格统一
- 大部分图在手机端侧栏阅读场景下不显著挤压
- 同类图视觉语法一致
- 图前导语与图内容匹配，不出现“图说的是 A，正文马上转去 B”

# 阶段 2：卷级边界与卷尾收口

**目标：** 让每一卷更像一本小书，而不是一组主题相关的草稿。

## 2.1 任务

### A. 逐卷检查卷首问题句
每卷都要能明确回答：
- 本卷到底在回答什么问题？
- 为什么这一卷必须存在？
- 它与上一卷、下一卷的边界是什么？

### B. 逐卷检查中段边界
重点检查高风险相邻章节：
- 卷三：01 / 02 / 03 / 04
- 卷五：01 / 02 / 03 / 04 / 05
- 卷六：01 / 02 / 03 / 04 / 05

### C. 重写或微调卷尾收口
每卷卷尾都要尽量压成一句 retained takeaway，例如：
- 卷三：Codex 持续存在，不靠查库拼对象，而靠重新接回一条还能继续推进的工作线
- 卷五：Codex 真正管理的不是命令，而是执行会话
- 卷六：Codex 正在长成更高层 runtime 组织，而不是只多了一些高级功能

### D. 统一卷级 index 的读法
每卷 index 需要统一具备：
- 卷问题
- 本卷读完后你会得到什么
- 目录
- 推荐阅读路径
- 卷间导流

## 2.2 产出物

- 逐卷卷首/卷尾修订
- 一份“卷级 retained takeaway 清单”
- 更新后的 volume index 页面

## 2.3 完成标准

- 每卷都能用一句话讲清存在意义
- 每卷卷尾都能留下一个稳定结论
- 高风险章节边界明显更清楚
- 读者从卷级 index 进入时，不会只看到文件列表，而能看到问题链

# 阶段 3：总入口、阅读地图与发布质量

**目标：** 让整本书对新读者真正可进入、可跳读、可传播。

## 3.1 任务

### A. 重写总入口页
总入口要更明确回答：
- 这本书整体在讲什么
- 为什么是 6 卷
- 什么读者适合怎么读
- 如果只关心 runtime / 恢复 / 控制面 / 执行 / 多 agent，该走哪条阅读路径

### B. 做一份阅读地图
至少应包括：
- 完整顺读路径
- runtime-first 路径
- execution-first 路径
- control-plane-first 路径
- high-runtime / agent-first 路径

### C. 做一张整本书总图
不是某一卷内部图，而是整本 guidebook 的认知递进图：
- 卷一：整机总图
- 卷二：怎么跑起来
- 卷三：怎么持续存在
- 卷四：怎么暴露控制面
- 卷五：怎么把动作变成执行会话
- 卷六：怎么长出更高层 runtime 组织

### D. 检查站点层体验
统一检查：
- 左侧 nav 是否完整
- 每卷 index 是否在导航里合理出现
- 首页与总入口是否重复
- 图是否在手机端显示正常
- 卷间跳转是否顺畅

## 3.2 产出物

- 新版总入口页
- 阅读地图页
- 整书认知递进图
- 站点导航和入口体验修订

## 3.3 完成标准

- 新读者不需要知道你的内部写作背景，也能理解该从哪里进入
- 总入口不只是“6 个链接”，而是“整本书的读法说明书”
- 站点入口、卷级入口、正文入口三层关系清晰

## 四、最后一轮：全书语言降噪

这个阶段不建议现在立刻做，但后面一定要做。

## 4.1 要解决的问题
- 某些地方术语密度太高
- 某些段落判断句连续堆叠
- 读者需要反复停下来解码
- 图下文字有时仍偏“作者脑中判断”，不够“读者可顺读”

## 4.2 处理原则
- 先问题，后机制
- 一段一个主判断
- 先低分辨率解释，再进入细节
- 把解释留在正文，不把太多说明挤进图里

## 4.3 完成标准
- 不只是“能读懂”
- 而是“读起来顺”
